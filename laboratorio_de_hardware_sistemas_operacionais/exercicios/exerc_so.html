1) Qual é a finalidade de um sistema operacional

a) Organizar a disposição dos dados em um dado elemento de memória
b) Compilar linguagem de programação para linguagem de máquina
c) Gerencia recursos do computador, oferece serviços para os programadores e distribui execução de outros programas.
d) Operacionalizar estruturas necessárias para que o usuário tenha acesso a internet

3) Sobre utilidade do sistema operacional, marque a verdadeira

a) Apesar do desenvolvimento de software ser algo complexo, o s.o não contribui em nada para facilidade dessa atividade
b) Não é papel do S.O esconder detalhes de hardware para o Programdor, logo essas questões devem ser pensadas por ele no momento de desenvolvimento do software
c) O SO oferece facilidades e serviços, como editores e depuradores, para auxiliar o programador na criação de programas.
d) O sistema operacional não atua em questões vinculadas a entrada e saída de dados

4) No que tange a Execução de Programas do S.O

O sistema operacional não tem responsabilidade nenhuma nisso
Antes do programa ser executado, o S.O o carrega em um memória de rápido acesso conhecida como ROM, hoje em dia disponibilizada fisicamente em discos rígidos
É impossível fazer algum tipo de execução de programas sem a disponibilidade de um sistema operacional
Diversas tarefas precisam ser realizadas para executar um programa. As instruções/dados precisam ser carregados para a memória, dispositivos de E/S precisam ser inicializados, etc. O SO trata de tudo isso para o usuário.

5) No que tange a Entrada e Saída

O sistema operacional gerencia apenas os dispositivos de entrada
Teclado é o exemplo de um dispositivo de saída
Nenhuma alternativa aqui presente é a correta
Monitor é o exemplo de um dispositivo de entrada

6) Sobre drivers assina-le a única errada

Sistemas operacionais contêm muitos drivers para controlar dispositivos de E/S.
Como cada tipo de controlador (controlador de memória, de E/S etc.) é diferente, diversos softwares são necessários para controlar cada um.
Drivers são programas usados para fazer a edição de códigos cuja finalidade é executar algum tipo de operação no kernel do sistema operacional
O software que conversa com um controlador, dando a ele comandos e aceitando respostas, é chamado de driver de dispositivo.

6) No que tange a relação de um sistema operacional e acessos ao sistema, marque a única errada

No caso de um sistema compartilhado ou público, o SO controla o acesso ao sistema e a seus recursos.
No caso de um sistema compartilhado ou público, o SO oferece proteção de recursos e dados de usuários não autorizados, e resolver conflitos para disputa de recurso
A adminsitração de tipos de acessos a um dado arquivo não é de responsabilidade do sistema operacional
O SO precisa tomar uma medida que encerre a condição de erro com o mínimo de impacto sobre as aplicações em execução, no caso deles ocorrem tem tempo de execução

7) No que tange ao Gerenciamento de Recursos

I) O conceito de um sistema operacional como fundamentalmente fornecendo abstrações para programas aplicativos é uma visão top-down (abstração de cima para baixo).
II) Uma visão alternativa, bottom-up (abstração de baixo para cima), sustenta que o sistema operacional está ali para gerenciar todas as partes de um sistema complexo.
III) Na visão bottom-up, a função do sistema operacional é fornecer uma alocação ordenada e controlada dos processadores, memórias e dispositivos de E/S entre os vários programas competindo por eles.
IV) O SO nunca abre mão do controle e precisa depender do processador para permitir que ele readquira o controle
V) O SO não funciona da mesma maneira que o software comum do computador; ou seja, ele não é um programa executado pelo processador.
